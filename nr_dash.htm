Approvals Process 05/25

This document explains the process for getting approval for setting MFA to be re-registered by the user or password resets through the IT helpdesk. The aim is to ensure all resets are properly authorised to maintain security.

Scope

This process applies to all requests for password and for setting MFA to be re-registered by the user for both individual and generic accounts within the organisation.

Approval Requirements

Individual Accounts

  • Manager Approval: Password and setting MFA to be re-registered by the user cannot be done without an email from the user's manager, sent to the approvals mailbox.
  • Alternate Approval: If the user's manager is unavailable, an email from an adjacent manager or a higher-level manager in the management chain is acceptable. i.e their managers manager, or another band 4 or above or manager adjacent to their direct manager. Use the org charts to check this.

Generic Accounts

  • Account Owner Approval: For generic accounts, the reset request must come from the account owner.
  • Route control Manager Approval: If the account owner is unavailable or it is during the night shift, the Route control manager, local operations manager, or mobile operations manager on shift can authorise the reset.

Process Flow

Request Submission

  1. User Request: The user contacts the IT helpdesk requesting a password or setting MFA to be re-registered.
  2. Ticket Creation: The IT helpdesk analyst creates a ticket in the system and provides the user with a reference number.

Approval Verification

  1. Manager Email: The user's manager (or alternate as specified) sends an email to the approvals mailbox authorising the reset.
  2. Generic Account Email: For generic accounts, the account owner (or alternate as specified) sends the authorisation email.

Reset Execution

  1. Approval Confirmation: The IT helpdesk analyst verifies the approval email.
  2. Reset Action: Upon confirmation, the analyst proceeds with the password or sets MFA to be re-registered.
  3. Notification: The user is notified of the reset completion and provided with any necessary instructions.

Special Considerations

  • Out-of-Hours Requests: For out-of-hours requests, the same approval process applies. If the manager or account owner is not available, the request will be processed during the next business hours.
  • Security Compliance: All resets must comply with the organisation's security policies and procedures.

Contacts iPhone to Android

Contacts iPhone to Android

Learn more
Yes, you can transfer contacts from an iPhone to an Android phone. The easiest method is by syncing your iPhone contacts to Google Contacts and then accessing them on your Android device. You can also export your iPhone contacts as a vCard file and import it to your Android.
Here's how to transfer contacts from iPhone to Android using Google Contacts:

  1. On your iPhone:
    • Open Settings and go to "Contacts".
    • Tap "Accounts".
    • Tap your existing Google account (or add a new one if you don't have one).
    • Toggle the "Contacts" switch to the "On" position.
    • This will start syncing your iPhone contacts with Google Contacts.
  2. On your Android phone:
    • Open the Contacts app.
Make sure you're signed in with the same Google account you used on your iPhone.
Your iPhone contacts should now appear in your Android Contacts app.
Alternatively, you can transfer contacts using a vCard file:
  1. On your iPhone:
    • Go to iCloud.com in a web browser (or use the iCloud app on your iPhone if you have it).
    • Log in to your iCloud account.
    • Select "Contacts".
    • Select all your contacts and then choose "Export vCard".
    • Download the vCard file to your iPhone.
  2. On your Android phone:
    • Open the Contacts app.
    • Find the "Import" or "Import from vCard" option.
    • Select the downloaded vCard file.
    • Your contacts should now be imported to your Android phone.

Zscaler Browser Isolation

Zscaler â Browser Isolation

You can recognise this by a pop up just underneath the address bar and by the URL which will start: https://32980588.isolation.zscaler.com/.

Browser isolation means that anyone accessing the link will not be able to:

  • Copy and paste to/ from the website.
  • Transfer files to/from the website
  • Print from the website
  • Input text to the website
  • Open attached office files
  • Local browser rendering is switched off.

If a user needs to have these functionalities added to a website, please check the website is legitimate, and has a 100% clean rating on https://www.virustotal.com/. If you are satisfied it is not malicious, then you can request it be recategorized from miscellaneous here: Site Review - URL Category Lookup Tool | Zscaler. Just enter the URL, suggest the correct category and provide the reasoning / evidence. The official response time is up to 14 days however security has informed us that they normally receive a response the next day.

The above will be created into a customer facing knowledge article for user to action themselves. Security have managed to manually go through a lot of the âmiscellaneousâ websites and whitelisted those which they believe to be trusted in order to cut down on inconvenience to users and calls to ourselves. If you do receive any calls please follow the process above which will be outlined in the Zscaler â Browser Isolation template.

For urgent enquiries, where a user requires same day access to the website, please complete the bottom half of the script and assign to security. They will be able to manually recategorise the link. Users MUST have a strong business justification for this to be actioned the same day.

Bios Update Failed

Steps to be taken by ITHD1:

  1. Unplug the power cable and disconnect external devices.
  2. Press and hold the power button for at least 15 seconds, this should turn off the machine
  3. Wait for 1 minute, and then press the power button to turn ON the machine.
  4. Machine should boot up normally and should present the login screen to the user.

User to Attemp:

  1. If the above steps do not work and the device is a laptop, keep the power cable disconnected and let the battery drain out so that device could turn off. Once device is turned off ,connect it back to the power supply , turn it ON and it should boot straight to Windows.

Steps to be taken via 2nd Line Supp:

  1. If BIOS update is showing âFailedâ in Software Center, please restart the machine and disconnect all the external devices except power cable and retry the BIOS update installation from Software Center.
  2. If above steps do not resolve the issue, send the ticket to HCL SCCM Scheduling queue to check this further.

If then passing a ticket to HCL, please add a note to advise all steps have been taken.
